# 理工科技术标书质量检查清单

## 1. 依据与条款

- [ ] 已读取招标正文、附表、响应表及澄清文件。
- [ ] 条款原文采用最终规范版本。
- [ ] “▲”“★”及否决性条款已单独核查。
- [ ] 条款编号、原文、指标和拟响应位置一致。
- [ ] 没有为了增加覆盖数量而强行标注。
- [ ] 同一条款没有无必要重复。

## 2. 数字与验收

- [ ] 数值、单位、符号、大小写和中英文空格统一。
- [ ] 测试时长、并发数、输入输出长度、准确率和可用性与招标文件一致。
- [ ] 没有加入自行换算的替代指标。
- [ ] 测试环境与实际交付环境一致。
- [ ] 被测模型版本、参数规模、量化方式和推理配置可核验。
- [ ] 测试集、训练集、验证集按要求隔离。
- [ ] 评分主体、评分规则、争议复核和证据材料明确。

## 3. 承诺风险

- [ ] 品牌、型号、版本、开源属性均有依据。
- [ ] 国产化或自主可控要求确实来自采购文件。
- [ ] SLA、响应时限、恢复时限和未达标处理已经责任团队确认。
- [ ] 满意度、训练频率、迭代周期等阈值有明确来源。
- [ ] 承诺函“已提供/将提供”的时态符合实际。
- [ ] 数据来源、授权、使用范围、删除义务和权属表述经法务核查。
- [ ] 没有无依据的“全量留存、自动剔除、不可篡改、永久保存”等承诺。

## 4. 任务边界

- [ ] 每段内容服务于当前课题和任务。
- [ ] 大模型、数据治理、视觉算法、系统建设和运维职责没有混写。
- [ ] 协同能力写明协同方式，不归为单项任务独立能力。
- [ ] 项目概要保持总述，服务范围强调做什么，技术路线强调怎么做。

## 5. 文风与结构

- [ ] 延续原标书的标题层级、段落方式和正式语气。
- [ ] 没有作者提示语、制图说明语或响应表填写提示。
- [ ] 没有孤立条款、空泛承诺和机械式总结。
- [ ] 长段已按核查逻辑合理拆分。
- [ ] 没有重复标题、错误编号或被意外拆开的段落。
- [ ] 专业名词准确且有实际作用。
- [ ] 宣传性和绝对化词语已降到必要程度。

## 6. 文件交付

- [ ] 文件名与用户要求完全一致。
- [ ] 字数接近目标且未通过重复内容凑字数。
- [ ] Markdown标题和正文编号符合指定格式。
- [ ] 原始DOCX未被意外覆盖。
- [ ] 已输出条款覆盖、格式、重复、偏题和字数自检结果。
