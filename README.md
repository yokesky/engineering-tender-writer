# engineering-tender-writer

一个用于撰写理工科技术标书的 AI 编码技能（Skill）。聚焦计算机、软件工程、网络空间安全、人工智能等领域，以及与电力系统、油田系统等工程场景融合的交叉领域实际项目。

本技能把招标要求转化为正式、可核查、可交付的技术标书文案，覆盖技术标书、服务方案、研究内容、建设方案、技术路线、模型或算法方案、性能验收、技术参数响应表与点对点响应等场景。

## 能力概览

- 条款台账：逐条记录招标要求，核对编号、参数性质、规范原文与当前覆盖情况，发现口径不一致先报告差异。
- 章节分工：区分项目概要、服务范围、研究内容、技术路线、建设方案、建设目标与应用落地逻辑，避免越界混写。
- 点对点响应：先写充分正文再嵌入条款，选择重点条款、硬指标与权属合规条款做代表性响应，拒绝覆盖数量竞赛。
- 承诺边界控制：未经招标依据不新增品牌、型号、版本、国产化限定、固定阈值或强实现承诺，识别风险并标注降调。
- 证据链构建：每个重要目标至少对应一种核查依据，确保测试集隔离、环境可复现、评测方法可复核。
- 完成终检：按质量检查清单逐项核查条款遗漏、数字一致性、任务边界、文风与文件交付。

## 目录结构

SKILL.md 是技能主文件，包含工作流、规则与交付格式。agents/openai.yaml 提供技能展示名称与默认提示。references 目录下有三份参考文档：quality-checklist.md 为标书质量检查清单，response-patterns.md 为点对点响应写法示例，workflow-templates.md 为条款台账、大纲与修改记录模板。

## 适用场景

- 从招标文件（DOCX/Markdown）中提取要求并核查条款覆盖；
- 编制技术标书正文、服务方案与技术路线；
- 生成并优化技术参数响应表与点对点响应；
- 审查现有标书草稿，识别无依据承诺与任务越界。
